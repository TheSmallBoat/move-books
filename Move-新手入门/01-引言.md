# 引言：什么是 Move

本章目标：让读者快速理解 Move 语言的设计目标与核心理念，为后续学习打好概念基础。

要点速览

- 设计目标：安全、可验证、用于表示与转移“资源”（资产）的语言。
- 资源模型：Move 将重要数据建模为不能被随意复制或丢弃的类型（资源），以便在类型系统层面保证资产安全性。
- 模块化：以 `module` 定义类型与函数，模块可以被编译与复用。
- 静态可验证：语言设计便于形式化验证（Move Prover），有助于写出更安全的合约代码。

为谁准备

- 目标读者：有一般编程经验，或有区块链背景但首次接触 Move。

快速示例（最小概念演示）

```move
module 0x42::example {
  struct Counter has key { i: u64 }

  public fun publish(account: &signer, i: u64) {
    move_to(account, Counter { i })
  }
}
```

说明：示例展示了声明一个 `key` 资源 `Counter`，以及把它发布到某个账户下的基本动作（此处为概念演示，部署/运行环境步骤在“快速上手”）。

阅读建议

- 按顺序学习：先掌握基本语法与变量、类型，再学函数、模块、资源与全局存储，最后看泛型、能力与规格证明。
- 实践优先：在本地用包工具或 REPL 编译并运行小示例，快速验证概念。
