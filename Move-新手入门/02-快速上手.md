# 快速上手（链无关）

目标：用最小依赖说明如何编写、编译与本地测试 Move 代码（不依赖特定链或 CLI）。

准备工作（最小）

- 安装 Move 编译器工具链（取决于你的环境），或使用仓库提供的在线/容器化工具。
- 将示例代码放在一个包（Move.toml + sources 目录）中，按章节示例编写 `.move` 源文件。

本地测试建议流程

1. 在工作目录创建 `Move.toml` 与 `sources/`。2. 将模块文件放入 `sources/`。3. 使用 `move` 工具或等价工具进行编译与单元测试（见下）。

示例（脚本+模块快速示范）

```move
script {
  fun main(x: u64) {
    let sum = x + 1; // 中文注：示例计算，将输入 x 增加 1
    // 中文注：在实际脚本运行环境中，可在此处调用断言或日志函数以检验结果
  }
}
```

单元测试（语言级别）

- 使用 `#[test]` 标注函数作为单元测试（见“单元测试”章节）。
- 将测试放在同一包内并在测试模式编译/运行。

注意

- 本手册移除了 Aptos 特定的运行/发布命令。
