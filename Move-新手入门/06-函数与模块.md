# 函数与模块

目标：讲清函数声明、可见性、参数、返回、`acquires` 注解与模块/包的基本概念。

函数要点

- 声明：`fun name<type_params>(args): ret { body }`。
- 可见性：默认模块私有；`public`、`package`、`public(friend)` 控制可访问范围。
- `entry`：标记为可作为模块入口调用（常用于链上调用语义）。
- `acquires`：当函数访问模块内的 `key` 资源时必须注明（确保引用安全）。

模块与脚本

- 模块：以 `module <addr>::name { ... }` 定义，包含类型、函数与常量。
- 脚本：一次性执行单元，通常调用模块的 `public` 函数。

示例

```move
module 0x42::example {
  struct Balance has key { value: u64 }
  struct Balance has key { value: u64 } // 中文注：资源类型，需放在全局地址中

  public fun deposit(s: &signer, v: u64) {
    move_to(s, Balance { value: v }) // 中文注：将 Balance 作为资源发布到签名者地址
  }

  public fun read(addr: address): u64 acquires Balance {
    borrow_global<Balance>(addr).value // 中文注：读取全局 Balance 的 value 字段
  }
}
```

实践建议

- 公共 API 设计时尽量保持函数签名简单并写好 `acquires`，以便后续形式化验证与安全审计。
- 使用 `inline` 谨慎：可以提高性能但可能增加字节码大小与复杂性。

## 原始示例（来自原文）

下面示例摘自原文，保留原代码以便验证语义：

```move
module 0x42::example {

  struct Balance has key { value: u64 }

  public fun add_balance(s: &signer, value: u64) {
    move_to(s, Balance { value }) // 中文注：向签名者地址发布 Balance 实例
  }

  public fun extract_balance(addr: address): u64 acquires Balance {
    let Balance { value } = move_from<Balance>(addr); // 中文注：从全局地址移除并取得资源字段（move out）
    value // 中文注：返回被移出的 value
  }
}
```
