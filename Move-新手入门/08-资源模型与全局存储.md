# 资源模型与全局存储

核心目标：理解资源（resource）如何映射到地址、全局存储 API 与安全性约束。

全局存储五大操作

- `move_to<T>(&signer, T)`：发布资源到 signer.address。
- `move_from<T>(address): T`：从地址移出资源并返回。
- `borrow_global<T>(address): &T`、`borrow_global_mut<T>(address): &mut T`：借用全局资源。
- `exists<T>(address): bool`：判断是否存在。

关键规则

- 资源类型必须带 `key` 能力才可用于全局读写。
- 访问资源的函数通常要标注 `acquires T`（编译器可在新版推断）。
- 不允许返回指向全局存储的引用（防止悬垂引用）。

示例（计数器）

```move
module 0x42::counter {
  struct Counter has key { i: u64 }

  public fun publish(account: &signer, i: u64) {
    move_to(account, Counter { i }) // 中文注：发布 Counter 资源到签名者地址
  }

  public fun get_count(addr: address): u64 acquires Counter {
    borrow_global<Counter>(addr).i // 中文注：借用全局资源并访问字段 i
  }
}
```

实践建议

- 在模块边界提供 CRUD 风格的 API（如 `publish`、`exists`、`increment`、`delete`），避免直接暴露字段。
- 小心跨模块依赖与资源升级兼容性（后续升级会有严格规则）。
