# 泛型与能力

本章目标：理解泛型类型/函数与能力（abilities）如何协同控制类型权能（如复制、存储、作为 key）。

泛型要点

- 函数与结构体可带类型参数：`fun id<T>(x: T): T { x }`。
- 调用时可显式写入类型参数或由编译器推断。
- 未被使用的类型参数可以是 phantom（见进阶）。

能力（四种）

- `copy`：允许被复制。
- `drop`：允许被丢弃（不必须显式处理）。
- `store`：允许出现在可持久化结构中。
- `key`：允许作为全局存储根（resource type）。

条件能力与泛型

- 泛型类型的能力依赖于其类型参数。例如 `vector<T>` 是否有 `copy` 取决于 `T` 是否有 `copy`。

示例（泛型结构）

```move
module 0x42::example {
  struct Cup<T> has copy, drop, store { item: T }

  fun use_cup() {
    let c = Cup<u64> { item: 42 }; // 中文注：用泛型类型创建实例，类型参数为 u64
    let c2 = copy c; // 中文注：由于 Cup 有 copy 能力，可直接复制 c
  }
}
```

注意

- 定义能力时，字段必须满足相应能力约束（例如声明 `has copy` 要求所有字段可 copy）。
- Phantom 参数与条件能力是高级特性，用于设计类型标签与避免不必要的能力传播。

## 原始示例（来自原文）

保留部分原始示例以便验证泛型与能力语义：

```move
module 0x42::example {
  fun id<T>(x: T): T {
    // this type annotation is unnecessary but valid
    (x: T) // 中文注：显式类型注解示例，通常可省略
  }
}
```

```move
module 0x2::m {
  // Currency Specifiers
  struct Currency1 {}
  struct Currency2 {}

  struct Coin<Currency> has store {
    value: u64
  }

  public fun mint_generic<Currency>(value: u64): Coin<Currency> {
    Coin { value } // 中文注：返回带类型参数的 Coin 实例
  }

  public fun mint_concrete(value: u64): Coin<Currency1> {
    Coin { value } // 中文注：返回具体货币类型的 Coin
  }
}
```
