# 单元测试与断言

目标：介绍 Move 的测试注解、`assert!` 宏与如何编写可靠的单元测试。

测试注解

- `#[test]`：标记为单元测试函数。
- `#[test_only]`：模块或成员仅在测试模式包含。
- `#[expected_failure]`：标记测试应当失败（可指定 abort_code）。

断言与中止

- `assert!(cond)`：条件不满足时 `abort`，可带 `u64` 代码或 `vector<u8>` 消息。
- `abort`：显式使执行失败并回滚状态。

示例（测试）

```move
module 0x1::my_module {
  struct MyCoin has key { value: u64 }

  public fun make_sure_non_zero_coin(coin: MyCoin): MyCoin {
    assert!(coin.value > 0, 0);
    coin
  }

  #[test]
  fun make_sure_non_zero_coin_passes() {
    let coin = MyCoin { value: 1 };
    let MyCoin { value: _ } = make_sure_non_zero_coin(coin);
  }
}
```

运行建议（本地）

- 在测试模式编译并运行测试。具体命令取决于你所用的 Move 工具链或包装脚本。

实践建议

- 用常量定义错误码便于维护（如 `const E_INDEX_OOB: u64 = 1;`）。
- 书写正/负例测试，结合 `#[expected_failure]` 测试边界情形。

## 原始示例（来自原文）

下面为来源于原仓库的完整测试模块示例，保留原代码以便验证测试语义（已移除与工具链无关的运行命令）：

```move
module 0x1::my_module {

  struct MyCoin has key { value: u64 }

  public fun make_sure_non_zero_coin(coin: MyCoin): MyCoin {
    assert!(coin.value > 0, 0); // 中文注：若 coin.value <= 0，则 abort 并返回错误码 0
    coin // 中文注：返回传入的 coin（所有权被移动/返回）
  }

  public fun has_coin(addr: address): bool {
    exists<MyCoin>(addr)
  }

  #[test]
  fun make_sure_non_zero_coin_passes() {
    let coin = MyCoin { value: 1 }; // 中文注：构造一个有效的 coin
    let MyCoin { value: _ } = make_sure_non_zero_coin(coin); // 中文注：调用应通过断言
  }

  #[test]
  #[expected_failure(abort_code = 0, location = Self)]
  fun make_sure_zero_coin_fails() {
    let coin = MyCoin { value: 0 }; // 中文注：构造一个无效的 coin
    let MyCoin { value: _ } = make_sure_non_zero_coin(coin); // 中文注：期望断言触发 abort
  }

  #[test_only] // test only helper function
  fun publish_coin(account: &signer) {
    move_to(account, MyCoin { value: 1 }) // 中文注：在测试中发布资源到签名者地址
  }

  #[test(a = @0x1, b = @0x2)]
  fun test_has_coin(a: signer, b: signer) {
    publish_coin(&a); // 中文注：在地址 a 发布 coin
    publish_coin(&b); // 中文注：在地址 b 发布 coin
    assert!(has_coin(@0x1), 0); // 中文注：断言 a 地址存在 coin
    assert!(has_coin(@0x2), 1); // 中文注：断言 b 地址存在 coin
    assert!(!has_coin(@0x3), 1); // 中文注：断言 c 地址不存在 coin
  }
}
```
