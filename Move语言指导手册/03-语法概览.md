# 03 语法概览：先认识 Move 代码长什么样

这一章不追求把语法一次讲完，而是先让你看懂 Move 代码的基本形状。你会发现，Move 的表达式风格很强：很多结构本身就是表达式，能返回值；块、分支和循环也经常和表达式组合在一起使用。

## 两种常见代码单元

Move 里最常见的两种顶层代码单元是 `module` 和 `script`：

```move
module 0x42::example {
    // 模块里通常定义类型、函数和常量
}

script {
    // 脚本通常只有一个入口函数
}
```

- `module` 更像可复用库。
- `script` 更像一次性执行片段。

## 表达式与块

很多 Move 语句其实都是表达式。块 `{ ... }` 的最后一个表达式往往就是整个块的结果。为了让你更容易阅读，Move 也允许在局部代码里写多条语句，再用最后一个值作为返回结果。

```move
script {
    fun demo(): u64 {
        let x = 10;
        let y = 20;
        x + y
    }
}
```

## 注释

Move 支持三种常见注释风格：

- `//` 单行注释
- `/* ... */` 块注释
- `///` 文档注释

文档注释写在类型、函数或模块声明上方，适合给未来的读者留下说明。对入门者来说，这点很重要：好的 Move 代码不只是“能运行”，还应该能让别人快速知道为什么这么写。

## 字面量和基本形状

你会经常看到这些写法：

- 整数字面量，如 `1`、`42u64`、`0xFF`。
- 地址字面量，如 `@0x42`。
- 布尔值 `true` 和 `false`。
- 字节串 `b"hello"` 和十六进制字节串 `x"48656C6C6F"`。

## 元组和单元

Move 有元组，但它们主要用于“多返回值”之类的场景。`()` 表示单元值，也就是“没有值”。这两个细节很容易混淆：

- `(a, b)` 是两个元素的元组。
- `()` 是空元组，也就是单元。
- `(a)` 只是 `a` 本身加了一层括号，不是单元素元组。

后面的章节会继续把整数、地址、向量、结构体和枚举这些语法形状拆开讲。

## 字面量推断的一个小例子

Move 很多字面量都依赖上下文推断类型。新手最常见的误区，是以为“写成 1 就一定是某个固定类型”。实际上，编译器会尽量从周围表达式推断：

```move
script {
    fun demo() {
        let n: u64 = 1;
        let m = 1;
        let total = n + m;
    }
}
```

在这类代码里，你最应该关注的不是数字写法本身，而是“编译器是从哪里知道类型的”。如果上下文不够明确，就需要你补上类型后缀或类型标注。这个习惯会在向量、函数返回值和泛型里反复出现。

## `if` 既能像语句，也能像表达式

原书里多次强调这一点：Move 的 `if` 不只是控制流转移，它也可以直接产生值。

```move
script {
    fun demo(x: u64, y: u64): u64 {
        let maximum = if (x > y) x else y;
        maximum
    }
}
```

中文备注：这里的 `if` 不是“执行完就结束”的控制语句，而是一个会返回 `u64` 的表达式。正因为它会返回值，所以两边分支的类型必须一致；否则编译器无法把这个表达式当成一个可用的值。

## `script` 和 `module` 的骨架

原书在介绍语言时，一开始就让你知道 Move 顶层代码一般长什么样。一个 `script` 通常由 `use`、常量和一个主函数组成；一个 `module` 则由类型、函数、常量、友元和导入构成。

```move
script {
    use std::debug;

    const ONE: u64 = 1;

    fun main(x: u64) {
        let sum = x + ONE;
        debug::print(&sum)
    }
}

module 0x42::example {
    use std::debug;

    const ONE: u64 = 1;

    struct Example has copy, drop { i: u64 }

    public fun print(x: u64) {
        let sum = x + ONE;
        let example = Example { i: sum };
        debug::print(&sum)
    }
}
```

中文备注：这个骨架非常重要，因为它告诉你“顶层结构和普通函数体不一样”。脚本通常只有一个主函数，模块则更像一个长期存在的命名空间和类型容器。

## 元组、单元和多返回值

Move 的元组主要用于多返回值和局部解构。原书特别提醒，`()` 是单元，不是某种“空的容器”。

```move
script {
    fun pair(x: u64, y: u64): (u64, u64) {
        (x, y)
    }

    fun demo() {
        let (a, b): (u64, u64) = pair(1, 2);
        let () = { };
    }
}
```

中文备注：如果你习惯了“括号就是分组”，这里最容易误读的是 `(e)`。它不是单元素元组，只是表达式分组。原书把元组和单元放在一起讲，就是为了避免这种常见误会。

## 标识符和路径

Move 里，名字不仅是名字，也是命名空间的一部分。`0x42::example::foo` 这种路径写法会在后面模块和函数章节里反复出现。对新手来说，最重要的是形成一个概念：标识符、地址和模块名不是分开的三种东西，它们是组合起来表达“这个名字属于谁”的。
