# 04 原始类型与变量：值、绑定与常量

这一章要解决两个问题：Move 里有哪些基础值类型，以及这些值是怎么被绑定到变量上的。你可以把它看成“认识数据”和“认识局部状态”的结合。

## 原始类型

Move 的基础类型主要有三类：整数、布尔值和地址。

### 整数

Move 提供多种整数类型，包括无符号整数 `u8`、`u16`、`u32`、`u64`、`u128`、`u256`，以及较新的有符号整数 `i8`、`i16`、`i32`、`i64`、`i128`、`i256`。

几个很重要的事实：

- 整数运算是受检查的，溢出、下溢、除零都会中止。
- 没写类型后缀时，编译器会尽量根据上下文推断；推不出来时通常会按 `u64` 处理。
- 可以用下划线提高可读性，比如 `1_000_000`。
- 负数写成 `-1i64` 这样的形式。

### 布尔值

`bool` 只有 `true` 和 `false` 两个值，常用于条件判断、断言和循环控制。

### 地址

`address` 用来表示账户或存储位置。它在底层是 256 位标识符，但在语言层面是透明的：你不能拿它做算术，也不能把它当普通整数随便处理。

## 局部变量与绑定

Move 使用 `let` 声明局部变量：

```move
script {
    fun demo() {
        let x = 1;
        let y = x + 2;
    }
}
```

你需要特别注意两点：

- 绑定必须先声明再使用。
- 赋值表达式 `x = value` 的结果是 `()`，不是被赋的那个值。

这意味着 Move 的赋值更像“改变已有局部变量的状态”，而不是“表达一个值再拿去继续计算”。

### 遮蔽

后面的 `let` 可以重新绑定同名变量，这叫遮蔽：

```move
script {
    fun demo() {
        let x = 1;
        let x = x + 1;
    }
}
```

遮蔽经常用来表达“在前一个值基础上得到一个新值”，读起来比频繁修改同一个变量更清楚。

### 复合赋值

如果你的工具链支持 Move 2.1 及以上，可以使用 `+=`、`-=` 这类复合赋值写法。它们只是语法糖，本质上仍然是“取旧值、计算新值、再写回去”。

## 常量

常量适合表示编译期就知道的共享值：

```move
const MAX_RETRY: u64 = 3;
const FLAG: bool = true;
```

常量的值必须能在编译时求出。它们很适合放错误码、固定地址、配置开关和一些不会变的边界值。

如果你使用的是较新的工具链，常量也可以带可见性修饰；但对初学者来说，先掌握“常量是编译期值”更重要。

## 先建立一个正确直觉

对新手来说，最容易犯的错误是把“变量”理解成“随便改的容器”。在 Move 里，变量只是局部绑定；真正重要的是值本身的类型，以及这个类型是否允许复制、丢弃和存储。后面的章节会继续把这个约束展开。

## 两个特别容易踩坑的细节

### 1. 向下转换可能中止

把大整数转换成小整数时，不是“自动截断”，而是会检查是否能放得下。这个规则很重要，因为它阻止你悄悄丢失高位数据。

```move
script {
    fun shrink(x: u64): u8 {
        x as u8
    }
}
```

中文备注：如果 `x` 超过 `u8` 能表示的最大值，转换就会中止。也就是说，`as` 不是 JavaScript 那种宽松转换，它是受检查的。

### 2. 局部变量必须先被真正赋值

Move 允许你先声明一个局部变量，再在后面某个分支里给它赋值。但如果所有路径都没有把它赋值完整，编译器就会报错。

```move
script {
    fun demo(cond: bool) {
        let x;
        if (cond) {
            x = 1;
        } else {
            x = 2;
        };
        let y = x + x;
    }
}
```

中文备注：这类写法的意义是“先留出位置，再在分支里补上值”。如果某个分支遗漏了赋值，`x` 就不会被视为已初始化。对习惯了宽松变量系统的读者来说，这一点一开始会不太自然，但它能显著减少未定义状态。

## 数字中的下划线

`1_000_000u64` 和 `1000000u64` 是同一个值，只是更容易读。这个细节很小，但在写错误码、阈值、计数和大整数常量时很实用。

## `move` 和 `copy`：变量语义的核心

原书在变量章节里非常强调这一点：Move 的值不是“默认随便复制”的。你必须显式说明某个值是被移动还是被复制。

```move
script {
    fun demo() {
        let x = 1u64;
        let y = copy x;
        let z = move x;
    }
}
```

中文备注：`copy` 和 `move` 的意义不只是语法上的区分，它们直接对应资源所有权。对普通整数来说，复制通常没有问题；但对资源值来说，移动意味着原值不能再被继续使用，这正是 Move 语言名字的来源。

## 算术、比较和逻辑运算

Move 对整数运算采用“受检查”语义：溢出、下溢、除零和非法位移都会中止。逻辑运算则是短路的。

```move
script {
    fun demo(a: u64, b: u64, p: bool, q: bool): bool {
        let sum = a + b;
        let diff = if (a > b) a - b else b - a;
        let cond = p && q || !p;
        cond
    }
}
```

中文备注：原书把这些规则讲清楚，是为了避免读者把 Move 当成“像大多数语言一样默认环绕”的整数系统。Move 的整数更像安全优先的受控整数：出错就中止，而不是悄悄给你一个看似合理、其实已经溢出的结果。

## 地址字面量和 `@`

`address` 是一个特殊的原始类型。它的字面量在表达式里需要加 `@`：

```move
script {
    fun demo() {
        let a: address = @0x42;
        let b: address = @0x1;
    }
}
```

中文备注：`@` 可以理解成把一个“命名空间项”转成“表达式项”的标记。原书在地址章节里特意强调这一点，是为了让你区分“地址值”和“地址名字”。这在命名地址、模块路径和交易参数里都很重要。

## 布尔值和短路

`bool` 只有 `true` 和 `false`，但它在控制流和断言里用得极多。

```move
script {
    fun demo(p: bool, q: bool): bool {
        p && q || !p
    }
}
```

中文备注：原书把布尔运算、条件表达式和 `assert!` 联系起来讲，是因为它们本质上都在帮你表达“某个条件是否成立”。在 Move 里，条件不仅决定分支，也经常决定事务能不能继续。

## 常量和内建常量

常量既可以是模块自己的共享值，也可以是语言内建的边界值，比如最大整数和测试编译标记。它们的意义在于：让“固定不变的规则”显式进入源码，而不是散落在代码里。
