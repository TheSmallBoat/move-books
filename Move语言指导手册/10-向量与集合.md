# 10 向量与集合：唯一的基础集合类型

Move 里最基础、最常用的集合类型是 `vector<T>`。它是一个同质、可增长、可缩短的序列。你可以把它理解为“Move 中唯一内建的动态数组”。

## 向量长什么样

向量既可以为空，也可以直接写元素：

```move
vector[]
vector[1, 2, 3]
vector<address>[@0x42, @0x100]
```

如果类型推断不够明确，也可以写成 `vector<T>[...]`。向量元素必须是同一种类型，而且元素不能是引用。

## `vector<u8>` 的特殊写法

`vector<u8>` 很常见，所以 Move 提供了更方便的字节串写法：

- `b"hello"`：字节串，支持常见转义。
- `x"48656C6C6F"`：十六进制字节串，每两个十六进制字符代表一个字节。

对做哈希、签名、错误消息或字节协议的人来说，这种写法会比手动列出每个字节舒服很多。

## 常用操作

`vector` 的操作主要来自标准库模块 `std::vector`。最常见的有：

- `empty<T>()`：创建空向量。
- `length`：查看长度。
- `push_back` / `pop_back`：在末尾追加或弹出。
- `borrow` / `borrow_mut`：借用某个位置的元素。
- `append`：把另一个向量拼接到末尾。
- `remove`、`swap_remove`：删除指定位置元素。
- `destroy_empty`：显式销毁空向量。

```move
script {
    use std::vector;

    fun demo() {
        let v = vector::empty<u64>();
        vector::push_back(&mut v, 5);
        vector::push_back(&mut v, 6);
        assert!(*vector::borrow(&v, 0) == 5, 0);
        assert!(vector::pop_back(&mut v) == 6, 0);
    }
}
```

## 下标语法

Move 2.0 以后，向量支持下标语法，写起来更接近常见数组操作：

- `&v[i]`
- `&mut v[i]`
- `v[i]`
- `v[i] = x`

这种语法本质上只是标准库借用操作的语法糖。

## 复制和销毁

向量能不能复制、能不能自动销毁，取决于元素类型的能力：

- 元素有 `copy`，向量才有 `copy`。
- 元素有 `drop`，向量才可以被自动丢弃。
- 元素没有 `drop`，你就必须显式处理它，通常要用 `destroy_empty`。

这正是 Move 设计的一贯风格：集合本身不会绕过元素类型的约束。

## 一个实用提醒

新手常常把向量当成“随便装东西的盒子”。在 Move 里它更像“受能力约束的序列”。它既能让你高效地管理多个值，也会严格保住元素的资源语义。

## 常用操作再补几个典型场景

```move
script {
    use std::vector;

    fun example() {
        let v = vector::empty<u64>();
        vector::push_back(&mut v, 5);
        vector::push_back(&mut v, 6);

        assert!(*vector::borrow(&v, 0) == 5, 42);
        assert!(*vector::borrow(&v, 1) == 6, 42);
        assert!(vector::pop_back(&mut v) == 6, 42);
        assert!(vector::pop_back(&mut v) == 5, 42);
    }
}
```

```move
script {
    use std::vector;

    fun example2() {
        let v = vector::singleton(7u64);
        assert!(vector::contains(&v, &7), 0);
        assert!(vector::index_of(&v, &7).0, 0);
    }
}
```

```move
script {
    use std::vector;

    fun example3() {
        let v = vector[1u64, 2u64, 3u64];
        let removed = vector::remove(&mut v, 1);
        let swapped = vector::swap_remove(&mut v, 0);
        let tail = vector::trim(&mut v, 0);
    }
}
```

中文备注：原书把 `remove`、`swap_remove`、`trim` 这些函数放进同一张表，就是在提醒你：集合操作不只是“增加”和“删除”，还包括是否保持顺序、是否复制回收值、是否显式处理被移出的元素。

## 下标语法和嵌套字段访问

```move
fun bubble_sort(v: vector<u64>) {
  use std::vector;
  let n = vector::length(&v);
  let i = 0;

  while (i < n) {
    let j = 0;
    while (j < n - i - 1) {
      if (v[j] > v[j + 1]) {
        let t = v[j];
        v[j] = v[j + 1];
        v[j + 1] = t;
      };
      j = j + 1;
    };
    i = i + 1;
  };
}
```

```move
module 0x42::example {
    struct S has drop { x: u64, y: u64 }

    fun example(v: vector<S>) {
        let x = v[0].x;
        let y_ref = &mut v[0].y;
        *y_ref = 10;
    }
}
```

中文备注：原书想说明的是，下标语法不是新的集合系统，而只是把 `borrow`、`borrow_mut` 和字段访问组合成更短的形式。理解语法糖背后的借用和解引用，才是关键。

## 销毁和复制向量的边界

```move
script {
    fun destroy_any_vector<T>(vec: vector<T>) {
        vector::destroy_empty(vec)
    }

    fun destroy_droppable_vector<T: drop>(vec: vector<T>) {
        // 什么都不写也可以
    }
}
```

```move
script {
    fun copy_vector<T: copy>(v: vector<T>): vector<T> {
        let v2 = copy v;
        v2
    }
}
```

中文备注：这部分的重点是，向量并不会绕过元素类型的能力约束。只有当元素本身可复制、可丢弃时，向量才会表现得像普通集合；否则，你就必须显式处理它们。

## 原书中最值得记住的几个向量例子

### 1. 字节串和十六进制字节串是同一类值的两种写法

```move
script {
    fun demo() {
        assert!(b"Hello!\n" == x"48656C6C6F210A", 0);
        assert!(b"\x48\x65\x6C\x6C\x6F" == x"48656C6C6F", 1);
    }
}
```

中文备注：原书之所以反复强调 `vector<u8>`，是因为它在字节协议、哈希、签名和错误消息里都很常见。你可以把 `b"..."` 看成“直接写字节内容”，把 `x"..."` 看成“按十六进制精确写字节”。

### 2. `destroy_empty` 不是多余的，它是在保护资源

```move
script {
    fun destroy_any_vector<T>(vec: vector<T>) {
        vector::destroy_empty(vec)
    }

    fun destroy_droppable_vector<T: drop>(vec: vector<T>) {
        // 如果 T 有 drop，就不需要显式销毁
    }
}
```

中文备注：这段是原书中非常典型的“能力驱动行为”例子。向量本身并不会偷偷替你处理资源；如果元素不能被随便丢弃，你就必须显式把它们处理干净。

### 3. 下标语法只是更短的写法

```move
fun bubble_sort(v: vector<u64>) {
    use std::vector;
    let n = vector::length(&v);
    let i = 0;

    while (i < n) {
        let j = 0;
        while (j < n - i - 1) {
            if (v[j] > v[j + 1]) {
                let t = v[j];
                v[j] = v[j + 1];
                v[j + 1] = t;
            };
            j = j + 1;
        };
        i = i + 1;
    };
}
```

中文备注：原书用这个例子说明，索引语法不是新能力，只是把 `vector::borrow` 和 `vector::borrow_mut` 包装成更自然的写法。真正重要的是，你读懂了它对应的借用和写回语义。
