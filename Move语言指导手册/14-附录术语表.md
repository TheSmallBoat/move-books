# 14 附录术语表：中英对照与快速回查

这份术语表不是把所有词都堆在一起，而是挑出你读 Move 时最常遇到、也最容易混淆的一批概念。建议你在阅读时保留英文术语，因为原书、错误信息和工具输出通常仍会使用它们。

## 核心术语

| 中文 | English | 简要说明 |
| --- | --- | --- |
| 模块 | module | 定义类型、函数和常量的代码边界。 |
| 脚本 | script | 一次性执行单元，通常调用模块公开函数。 |
| 资源 | resource | 需要显式追踪所有权的值。 |
| 能力 | ability | 决定值是否可复制、可丢弃、可存储、可作为全局根。 |
| 全局存储 | global storage | Move 持久保存模块和资源的地方。 |
| 签名者 | signer | 表示账户授权的值，常用于资源发布。 |
| 入口函数 | entry | 可以作为事务入口调用的函数。 |
| 可见性 | visibility | 决定函数、类型或常量对外开放的范围。 |
| 友元 | friend | 被允许访问受限接口的模块。 |
| 包 | package | 一组可构建、可复用的 Move 源代码。 |
| 类型参数 | type parameter | 泛型中抽象出来的类型占位符。 |
| 类型实参 | type argument | 调用泛型时实际填入的类型。 |
| 向量 | vector | Move 唯一的基础集合类型。 |
| 借用 | borrow | 以引用形式暂时访问某个值。 |
| 断言 | assert | 用条件检查程序前提或不变量。 |
| 中止 | abort | 终止事务并回滚全局状态修改。 |
| 规格语言 | MSL | Move Specification Language，用于描述程序行为。 |
| 证明器 | Move Prover | 静态检查规格与程序一致性的工具。 |

## 常见提醒

- `copy` 不等于“随便复制对象”，它是类型能力。
- `drop` 不等于“释放内存”，它是类型能力。
- `store` 不等于“能保存到文件”，它是存储能力。
- `key` 不等于“主键”，它是全局存储根能力。

## 阅读建议

当你在原书、报错信息或编译器提示里看到这些词时，最好优先按本表理解，再去结合上下文。这样能减少把普通编程语义误套到 Move 上的情况。

## 容易混淆的概念

| 概念 | 容易误解成 | 更准确的理解 |
| --- | --- | --- |
| `copy` | 普通语言里的自动复制 | 一种类型能力，只有满足条件时才能复制值。 |
| `drop` | 析构函数或自动释放内存 | 一种类型能力，表示值可以被丢弃。 |
| `store` | 任意持久化保存 | 一种类型能力，表示值可以作为结构体字段进入全局存储。 |
| `key` | 数据库主键 | 一种类型能力，表示值可以作为全局存储根。 |
| `entry` | 只能外部调用一次 | 可以作为事务入口，但也可能被别的 Move 函数调用。 |
| `package` | 运行时包权限 | 编译期包内可见，和 friend 语义相关。 |
| `friend` | 完全公开 | 只对明确声明的友元模块开放。 |
| `abort` | 抛异常 | 终止整个事务并回滚全局状态修改。 |
| `assert!` | 普通函数 | 编译器提供的宏式断言，失败时会中止。 |
| `acquires` | 访问权限声明 | 声明函数会接触哪些全局资源，便于静态检查。 |
| `MSL` | 运行时代码 | Move 的规格语言，用于描述和验证程序行为。 |
| `lemma` | 普通函数 | 只用于规格和证明的可复用定理。 |

## 一句总结

如果你在原书里只记住一个原则，那应该是：Move 的大多数设计都在把“资源安全”和“行为可验证”前置到类型系统和规格系统里。这个原则会贯穿整本书的每一章。
