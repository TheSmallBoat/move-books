# Move语言指导手册

这本手册不是《The Move on Aptos Book》的逐句翻译，而是把原书的知识点重新组织成一套更适合中文读者的入门材料。写作目标有三点：尽量保留原书中准确而有价值的事实性内容；在原书例子的基础上补足中文说明、提醒和上下文；用统一、友好的叙述方式帮助新手真正理解 Move 的设计思想。

你可以把它当作一份“从语义出发”的 Move 讲义，而不是命令手册。书中的示例尽量沿用原书语义，但会用更直接的中文解释来说明为什么这样写、这样用，以及哪些地方最容易出错。

## 这本手册的阅读方式

- 如果你刚接触 Move，建议按章节顺序阅读，从引言到测试与规格，再到实践规范。
- 如果你已经写过其他语言，可以把 03 到 10 章当作主体，重点抓住 Move 的类型系统、资源模型和全局存储。
- 如果你关心审计、验证或规范，12 章会给出 Move Prover 和规范语言的最小可用视角。

## 写作原则

- 先讲结论，再补理由，最后给例子。
- 对新手不友好的地方会直接指出，例如资源不能随意复制、全局存储引用不能逃逸、`assert!` 和 `abort` 的事务语义。
- 版本差异会尽量标出来，避免把旧版本语法和新版本特性混在一起。
